# Бриф Модуля: Sync And Storage

## Назначение

Этот модуль отвечает за хранение данных на каждом узле и их последующую синхронизацию.

## Что хранится локально

- настройки;
- логи;
- пресеты;
- сценарии;
- статусы и служебные снимки состояния.

Подтвержденный storage baseline:

- на `ESP32` тяжелые данные уходят на `SD card module` как SPI-расширение памяти;
- на `Raspberry Pi` heavy content хранится в локальной файловой системе;
- `ESP32 SD` также должна уметь принимать резервные synced-файлы с turret-side.

## Что синхронизируется

- пользовательские настройки;
- язык и тема;
- сценарии;
- статусы модулей;
- события и логи;
- capability-информация.
- media/report артефакты и metadata для `Gallery > Reports`.

## Что важно

- нельзя молча терять журналы;
- нельзя позволять удаленному узлу снимать локальный fault без правил;
- нельзя создавать запутанный "полу-мастер" режим без владельца.
- нельзя считать `ESP32 LittleFS` основным хранилищем для тяжелых ассетов и зеркальных копий;
- нужно сохранять origin/source-owner metadata при копировании файлов между узлами.

## Первые задачи

1. Описать формат журналов.
2. Описать формат heartbeat и статуса узла.
3. Описать синхронизацию настроек и сценариев.
4. Зафиксировать правила merge для журналов.
5. Зафиксировать правила резервного приема turret-файлов на `ESP32 SD`.
6. Зафиксировать metadata contract для `Gallery > Reports` и mirrored content.
