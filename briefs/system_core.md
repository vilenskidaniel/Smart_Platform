# Бриф Модуля: System Core

## Назначение

`System Core` — это клей новой платформы. Он не управляет конкретным железом, а задает общую модель статусов, режимов, блокировок и видимости модулей.

## Зона ответственности

- общий статус системы;
- состояния узлов;
- модель доступности модулей;
- приоритет режимов;
- аварийные и fault-состояния;
- общие capability-флаги.

## Что модуль обязан отдавать

- `system snapshot`
- `node health`
- `module registry`
- `active mode`
- `global fault summary`

## Что модуль не должен делать

- напрямую управлять GPIO;
- знать детали полива, турели или строба;
- выполнять сценарии конкретного модуля.

## Первые задачи

1. Описать структуры статуса.
2. Описать единые состояния модулей.
3. Описать глобальные stop/fault причины.
4. Подготовить базовый API для shell UI.
